LAKEBASE REMAINS THE SOURCE OF TRUTH FOR OPERATIONS
Delta remains the source of truth for analytics and ML training. Lakebase is the operational bridge — it handles real-time queries, bidirectional operator updates via native JDBC, ACID transactions across recommendation + work order + equipment status tables, and push notifications. Operator decisions sync back to Delta asynchronously every 5 minutes.

Java Module · Installed on gateway
→ Subscribes to Ignition tag engine
→ Binary-serialises tag change events
→ Forwards via secure outbound socket
→ No Kafka, no broker, no middleware
→ Works with any OPC-UA / tag source
→
DATABRICKS PLATFORM
Zerobus Service
Runs on Databricks · Receives stream
→ Deserialises incoming tag events
→ Writes directly to Delta Lake
→ Sub-second latency end-to-end
→ Triggers downstream DLT pipeline
→ Unity Catalog governed from day 1
→
ENRICHMENT SOURCES
Delta Live Tables
Merged into unified Silver layer
◈ OSI PI / Historian — PI Lakeflow connector
◈ CMMS / Maintenance — work orders, asset records
◈ ERP / SAP — production schedules, BOM
All joins happen in DLT — no data movement outside Databricks
KEY DIFFERENTIATOR
Zerobus eliminates the Kafka/MQTT broker layer entirely. The Java module on the Ignition gateway ships tag change events directly to the Databricks-hosted service — Ignition data is in Delta Lake within milliseconds, enriched with historian, maintenance, and ERP context in a single DLT pipeline.
