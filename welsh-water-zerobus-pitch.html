OPC-UA via MQTT relay
ScopeX OPC-UA data is currently serialised into MQTT and decoded by the OPC-UA Handler Azure Function. The connector supports native OPC-UA, removing that relay step.
Multi-service ingest chain
The current path spans five Azure services between Mosquitto and Delta Lake. The connector consolidates protocol handling and buffering into a single process in the OT DMZ.
Technolog via AWS IoT Core
Technolog telemetry currently routes through AWS IoT Core before reaching Azure. Subject to contract ownership, this path can be consolidated into the Mosquitto broker directly.
Connector Overview
Unified OT Zerobus Connector
Native MQTT subscriber Subscribes directly to Ovarro Mosquitto. Handles LUCID, Logger OPC-UA, and NRW OPC-UA topic groups in a single process. Sparkplug B auto-detection included.
Native OPC-UA client Connects directly to OPC-UA servers. Full hierarchical browse-path capture with vendor format detection for ISA-95 normalisation. Supports SignAndEncrypt security mode.
Zerobus gRPC egress Writes directly to Databricks Delta tables over gRPC/protobuf. Per-source routing to separate Delta tables configurable per source in a single deployment.
Backpressure and disk spooling Memory queue with automatic disk overflow on network congestion. Store-and-forward on the connector preserves data across transient connectivity interruptions.
Docker-native, OT DMZ ready Single container. No Ignition Gateway required. Deployable alongside an existing Mosquitto broker. Web UI on :8080 with Prometheus metrics for operational visibility.

With Unified Connector
–One Docker container in OT DMZ subscribes all three topic groups, writes to Delta Lake directly via Zerobus gRPC
–Direct OPC-UA to ScopeX — native hierarchical path capture, no serialisation step required
–Per-source Delta routing — lucid, logger, nrw each land in separate Raw tables in welsh_water catalog
–ISA-95 normalisation at ingestion — asset hierarchy applied before DLT Bronze processing
–NRW REST API path unchanged — Azure Function to SQL MI for public internet pull remains as-is
–Azure Databricks Prep and Train unchanged — DLT continues Bronze to Gold as designed
